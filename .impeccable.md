# Everthorn — Impeccable Design Context

## Design Context

### Users
Mixed-age Minecraft Bedrock players spanning casual gamers to dedicated community members. The audience includes prospective applicants discovering Everthorn for the first time, active players checking quests and the world map, and admins managing the community. Emotional context ranges from **excitement and wonder** on the landing page to **focused productivity** in the admin dashboard. The 7-year community history evokes **nostalgia and belonging** — visitors should feel they're joining something enduring and well-cared-for.

### Brand Personality
**Crafted. Nostalgic. Communal.**

Everthorn is a mature, long-standing community (7+ years) that values quality and permanence. The brand voice is warm and inviting without being saccharine — like a well-loved server that's always there when you need it. The interface should evoke the feeling of returning to a familiar place that has been thoughtfully maintained and evolved over time.

- **Voice:** Friendly, knowledgeable, unhurried. Speaks to players as peers, not customers.
- **Tone:** Welcoming and confident. Never corporate, never childish.
- **Emotional goals:** Belonging, wonder, trust, quiet pride in being part of something lasting.

### Aesthetic Direction

**Dual-mode design identity:**

| Surface | Direction | Reference |
|---------|-----------|-----------|
| **Public-facing** (landing, guidelines, events, map) | Bold, cinematic, community-driven | Gaming studio landing pages — immersive hero sections, strong visual narratives, motion-rich |
| **Admin** (dashboard, quests, projects, map editor) | Clean, productive, functional | Stripe-style — dense information, precise spacing, minimal chrome, fast-feeling |

**Theme:** Dark mode preferred, light mode fully supported.

**Visual tone:**
- Neutrals-first palette (shadcn neutral base) with selective color accents
- Rainbow/pastel gradient as the signature Everthorn brand element (used sparingly for CTAs, text highlights, and celebratory moments)
- Custom Minecraft pixel fonts (Minecraft Ten, Minecraft Seven) for headings on public pages — NOT for admin UI
- Inter (weight 300 light) for body text throughout
- Glassmorphism on sticky headers (backdrop-blur + semi-transparent backgrounds)
- Rounded corners (10px base radius) — soft but not bubbly
- Motion-rich but purposeful: confetti for celebrations, smooth page transitions, animated gradients

**Anti-references (what this should NOT look like):**
- ❌ Corporate/sterile SaaS — no cold blue-grey dashboards, no stock photo aesthetics
- ❌ Overly childish/cartoonish — despite Minecraft being family-friendly, the UI must feel mature
- ❌ Dark/moody gaming sites — no neon-on-black, no aggressive "hardcore gamer" aesthetic
- ❌ Generic AI-generated UI — no identical card grids, no gradient text used as a crutch, no glassmorphism everywhere

### Design Principles

1. **Crafted, not templated.** Every page should feel intentional. Reuse components consistently, but never let the UI feel like it was assembled from a kit. Custom touches (Minecraft fonts, rainbow gradients, custom animations) signal that this site was built *for* this community.

2. **Public pages inspire, admin pages empower.** The landing page and community-facing content should be cinematic and emotionally engaging — large imagery, bold typography, motion. The admin interface should be razor-efficient — dense, precise, fast. Never mix these modes.

3. **Dark-first, light-ready.** Design primarily in dark mode. Light mode is a well-supported alternate, not an afterthought, but dark is the native experience. Ensure all colors have sufficient contrast in both modes (WCAG 2.1 AA).

4. **Motion tells stories, not just decorates.** Animations should communicate state changes, guide attention, and create delight. Never animate for the sake of animation. Follow the transform/opacity rule — no layout-property animations. Use exponential easing for natural feel.

5. **Consistency is the brand.** The rainbow gradient, the Minecraft fonts, the glass headers, the badge color system — these are recognizable Everthorn signatures. Guard them. Before creating new visual patterns, check existing components. When a new pattern is needed, make it reusable.

## Technical Design Tokens

### Typography
| Token | Value | Usage |
|-------|-------|-------|
| `--font-sans` | Inter | Body text, UI chrome |
| `--font-mono` | Source Code Pro | Code blocks |
| `--font-minecraft-ten` | Minecraft Ten (custom .woff) | Display headings (public only) |
| `--font-minecraft-seven` | Minecraft Seven (custom .woff2) | Subheadings, labels (public only) |
| Body weight | 300 (light) | Default body text |
| Heading weight | 700 (bold) | All h1-h6 |
| Heading tracking | -0.025em (tight) | All h1-h6 |

### Heading Scale
| Level | Size |
|-------|------|
| h1 | text-4xl → md:text-5xl → lg:text-6xl |
| h2 | text-3xl → md:text-4xl → lg:text-5xl |
| h3 | text-2xl → md:text-3xl |
| h4 | text-xl → md:text-2xl |
| h5 | text-lg → md:text-xl (semibold) |
| h6 | text-base → md:text-lg (semibold) |

### Colors (OKLCH)
- **Base:** shadcn neutral palette
- **Dark sidebar accent:** `oklch(0.90 0.38 365)` (reddish-pink)
- **Glow accent:** `rgba(249, 115, 22)` (orange/amber) — used for text glow animation
- **Rainbow gradient:** `#ecd4ff → #ffd9c4 → #fff9d4 → #d4ffd4 → #d4f4ff → #d4dcff → #e4d4ff → #ffd4eb`

### Spacing
- **Base unit:** 4px (Tailwind default)
- **Standard gaps:** gap-2 (8px), gap-3 (12px), gap-6 (24px)
- **Use `gap-*` not `space-y-*`** for layout consistency

### Border Radius
- **Base:** `--radius` = 0.625rem (10px)
- **sm:** 6px, **md:** 8px, **lg:** 10px, **xl:** 14px

### Layout
- **Navbar height:** 3.5rem (56px)
- **Sidebar expanded:** 16rem (256px)
- **Sidebar collapsed:** 3rem (48px)
- **Sidebar mobile:** 18rem (288px)

### Motion
- **Standard transition:** `transition-all duration-200`
- **Glow text animation:** 2s infinite ease-in-out
- **Rule:** Transform and opacity only — never animate layout properties

### Accessibility
- **WCAG 2.1 AA** minimum for all color contrast
- **Keyboard navigable** — all interactive elements reachable and focusable
- **Screen reader support** — semantic HTML, ARIA labels where needed
- **Reduced motion** — respect `prefers-reduced-motion` media query

## Component Conventions

| Element | Convention |
|---------|------------|
| Buttons | 7 variants, 6 sizes, rounded-md |
| Badges | 20+ color variants, `text-xs font-bold`, rounded-md |
| Cards | `rounded-xl border shadow-sm bg-card`, gap-6, px-6 |
| Forms | FieldGroup + Field system from `@/components/ui/field` |
| Icons | Phosphor (primary), Lucide (secondary) |
| Sticky headers | `bg-background/50 backdrop-blur-sm` |
| Glass elements | Semi-transparent bg + backdrop-blur |
